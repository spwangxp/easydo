### 1.2.2 PipelineRun 模型修正版 (easydo-server/internal/models/pipelinerun.go)

**修正说明**：每次执行需要保存配置快照，日志结果绑定到子任务

```go
type PipelineRun struct {
    BaseModel
    PipelineID   uint64 `gorm:"index;not null" json:"pipeline_id"`
    BuildNumber  int    `gorm:"not null" json:"build_number"`
    Status       string `gorm:"size:32;not null" json:"status"`  // pending/running/success/failed/cancelled
    TriggerType  string `gorm:"size:32" json:"trigger_type"`     // manual/webhook/schedule
    TriggerUser  string `gorm:"size:64" json:"trigger_user"`
    TriggerSource string `gorm:"size:256" json:"trigger_source"` // webhook来源URL
    StartTime    int64  `json:"start_time"`
    EndTime      int64  `json:"end_time"`
    Duration     int    `json:"duration"`
    ErrorMsg     string `gorm:"type:text" json:"error_msg"`      // 失败时的错误信息
    
    // 关键设计：保存执行时的配置快照（因为流水线可能被编辑）
    // 每次执行使用独立的配置副本，互不影响
    Config       string `gorm:"type:longtext" json:"config"`  // 执行时的 PipelineConfig JSON 快照
    
    // 执行时绑定的 Agent（一次 PipelineRun 只使用一个 Agent）
    AgentID uint64 `gorm:"index" json:"agent_id"`
    
    // 关联
    Pipeline *Pipeline   `gorm:"foreignKey:PipelineID" json:"pipeline"`
    Agent    *Agent      `gorm:"foreignKey:AgentID" json:"agent"`
    Tasks    []AgentTask `gorm:"foreignKey:PipelineRunID" json:"tasks"`  // 所有子任务
}
```

**设计说明**：

| 字段 | 必要性 | 原因 |
|------|--------|------|
| Config | **必须** | 流水线配置会被编辑，需要保存执行时的快照 |
| AgentID | **必须** | 记录本次执行使用的 Agent，便于问题追溯 |
| Tasks | **必须** | 一对多关系，关联所有子任务 |

**为什么需要配置快照**：

```
时间线：
T1: Pipeline #1 配置为 [A → B → C]，执行 Build #1
    ↓ PipelineRun #1.Config = "A→B→C"

T2: 用户修改 Pipeline #1 配置为 [A → D → C]，执行 Build #2
    ↓ PipelineRun #2.Config = "A→D→C"

T3: 查看 Build #1 的执行记录
    → 应该显示 "A→B→C"（当时执行的配置）
    → 而不是当前流水线配置 "A→D→C"
```

**关键修正**：
- 移除 PipelineRun 的强制 Branch/CommitID 字段
- 改为可选的 Default 前缀字段
- 实际的代码仓库信息应该存储在 PipelineConfig.Nodes 中
```

#### 1.1.5 AgentLog 模型（执行日志）

```go
// AgentLog 记录任务执行的日志（任务完成后批量写入）
type AgentLog struct {
    BaseModel
    TaskID    uint64 `gorm:"index;not null" json:"task_id"`  // 关联任务（必填）

}
```

**设计说明**：

| 字段 | 说明 | 状态 |
|------|------|------|
| TaskID | 关联任务（必填） | ✅ 保留 |
| Level | 日志级别 | ✅ 保留 |
| Message | 日志内容 | ✅ 保留 |
| Timestamp | 时间戳 | ✅ 保留 |
| Source | 输出源 | ✅ 保留 |

**移除的字段**：

| 字段 | 原因 |
|------|------|
| PipelineRunID | 冗余，可通过 Task.PipelineRunID 关联 |
| LineNumber | 流式日志行号不可靠，移除 |

**WebSocket 频道设计**：

| 频道 | 订阅方 | 用途 |
|------|--------|------|
| `run:{run_id}` | Frontend | 订阅单个流水线运行的所有日志和状态 |
| `run:{run_id}:task:{node_id}` | Frontend | 订阅单个任务的日志（更精细） |
| `agent:{agent_id}` | Frontend | 订阅 Agent 状态更新 |

**刷新时机（可配置，默认 5 秒）**：

| 事件 | 刷新触发 | 说明 |
|------|---------|------|
| 定时器 | 每 5 秒（默认） | 周期性刷新缓冲队列到 MySQL |
| 子任务完成 | 立即刷新 | 任务结束时立即持久化剩余日志 |
| 异常情况 | 补偿机制 | 断连后重连时检查并刷新 |

#### 1.1.6 执行快照设计

**设计原则**：每次 PipelineRun 执行时，保存完整的配置快照和任务参数快照。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        执行快照设计                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Pipeline (流水线模板，可编辑)                                            │
│  ├─ id: 1                                                                │
│  ├─ name: "CI Pipeline"                                                 │
│  └─ config: "A → B → C"  (当前最新配置)                                  │
│                                                                          │
│  PipelineRun #1 (Build #1，执行时的快照)                                  │
│  ├─ config_snapshot: "A → B → C"  (执行时复制一份)                       │
│  ├─ AgentID: 10                                                         │
│  └─ Tasks:                                                               │
│      ├─ Task #1 (Node A 的快照)                                          │
│      │   ├─ NodeID: "1"                                                 │
│      │   ├─ Script: "npm install"  (执行时的脚本)                        │
│      │   ├─ Status: "success"                                           │
│      │   └─ Logs: [AgentLog #1, AgentLog #2, ...]                        │
│      │                                                                │
│      ├─ Task #2 (Node B 的快照)                                          │
│      │   ├─ NodeID: "2"                                                 │
│      │   ├─ Script: "npm run build"                                     │
│      │   └─ Status: "success"                                           │
│      │                                                                │
│      └─ Task #3 (Node C 的快照)                                          │
│          ├─ NodeID: "3"                                                 │
│          ├─ Script: "echo done"                                         │
│          └─ Status: "success"                                           │
│                                                                          │
│  PipelineRun #2 (Build #2，流水线已修改)                                  │
│  ├─ config_snapshot: "A → D → C"  (新的配置快照)                         │
│  └─ Tasks:                                                               │
│      ├─ Task #4 (Node A)                                                │
│      ├─ Task #5 (Node D)  ← 新增的任务                                   │
│      └─ Task #6 (Node C)                                                │
│                                                                          │
│  关键点：                                                                │
│  • PipelineRun.Config 是执行时的快照，不受后续 Pipeline 编辑影响          │

**查询执行历史**：

```go
// 获取某次流水线执行的完整信息
func GetPipelineRunDetail(runID uint64) (*PipelineRun, error) {
    var run PipelineRun
    err := DB.Preload("Agent").
            Preload("Tasks").
            Preload("Tasks.Logs", "level >=", "info").  // 只加载 info 及以上级别日志
            Preload("Tasks.Executions").
            First(&run, runID).Error
    return &run, err
}

// 获取任务的完整日志（按时间排序）
func GetTaskLogs(taskID uint64) ([]AgentLog, error) {
    var logs []AgentLog
    err := DB.Where("task_id = ?", taskID).
            Order("timestamp ASC").
            Find(&logs).Error
    return logs, err
}
```

| | PipelineRunID | ✅ 必须 | 关联所属的流水线运行记录 |
| | NodeID | ✅ 必须 | 关联 PipelineConfig 中的节点配置 |
| **标识字段** | TaskType | ✅ 必须 | 任务类型（shell/git_clone/build 等），决定执行器选择 |
| | Name | ✅ 必须 | 任务名称，用于 UI 展示 |
| **快照字段** | Params | ✅ 必须 | 执行时的参数快照，防止后续配置变更影响历史记录 |
| | Script | ✅ 必须 | 执行脚本快照，同上 |
| | WorkDir | ✅ 可选 | 工作目录快照，便于调试和审计 |
| | EnvVars | ✅ 可选 | 环境变量快照，便于问题复现 |
| **状态字段** | Status | ✅ 必须 | 任务状态（pending/running/success/failed） |
| | Priority | ✅ 可选 | 调度优先级，支持重要任务优先执行 |
| | Timeout | ✅ 必须 | 超时时间，防止任务无限挂起 |
| | RetryCount | ✅ 必须 | 当前重试次数 |
| | MaxRetries | ✅ 必须 | 最大重试次数限制 |
| | ExitCode | ✅ 必须 | 退出码，0 表示成功 |
| | ErrorMsg | ✅ 必须 | 错误信息，便于排查问题 |
| | StartTime | ✅ 必须 | 开始时间，用于统计和展示 |
| | EndTime | ✅ 必须 | 结束时间 |
| | Duration | ✅ 可选 | 缓存的执行时长，避免每次计算 |
| **结果字段** | ResultData | ✅ 必须 | JSON 格式的输出结果（outputs） |
| **仓库字段** | RepoURL | ✅ 必须 | 代码仓库地址，每个任务可能对应不同仓库 |
| | RepoBranch | ✅ 必须 | 分支信息 |
| | RepoCommit | ✅ 必须 | 提交 ID，用于版本追溯 |
| | RepoPath | ✅ 必须 | 本地检出路径 |

**设计原则说明**：

1. **快照字段的必要性**：
   - `Params` 和 `Script` 是**执行时快照**，而非冗余数据
   - 流水线配置可能被编辑，但历史运行的执行记录不应改变
   - 这与 PipelineRun.Config 保存配置快照的原理一致

2. **仓库信息的必要性**：
   - 单个流水线可能包含来自**不同仓库**的任务
   - 前端仓库和后端仓库可能使用不同的分支
   - 每个任务的仓库信息是执行时的真实上下文

3. **派生字段的取舍**：
   - `Duration` 虽然可通过 `EndTime - StartTime` 计算，但缓存后查询性能更好
   - `WorkDir` 和 `EnvVars` 主要用于审计和调试，保留有助排查

**与其他模型的关系**：

```
PipelineRun (1) ─────▶ (N) AgentTask (1) ─────▶ (N) TaskExecution
                         │
                         ├──▶ (N) AgentLog
                         │
                         └──▶ (1) Agent
```

- **PipelineRun**: 保存配置快照和整体状态
- **AgentTask**: 保存任务级别的执行快照
- **TaskExecution**: 记录每次重试的详细信息
- **AgentLog**: 记录实时日志（现在通过 Redis 缓冲 + 定时刷新）

**日志和结果绑定说明**：

| 关联表 | 作用 | 查询方式 |
|--------|------|----------|
| TaskExecution | 记录每次重试的执行信息 | Preload("Executions") |
| AgentLog | 记录实时日志（stdout/stderr） | Preload("Logs") |

### 1.2 当前执行逻辑 (handlers/pipeline.go)

**executePipelineTasks 方法 (第407-481行)**:

```go
func (h *PipelineHandler) executePipelineTasks(pipeline models.Pipeline, run *models.PipelineRun) {
    var config PipelineConfig
    json.Unmarshal([]byte(pipeline.Config), &config)

    // 构建节点映射
    nodeMap := make(map[string]*PipelineNode)
    for i := range config.Nodes {
        nodeMap[config.Nodes[i].ID] = &config.Nodes[i]
    }

    // 简单的BFS执行（按依赖顺序）
    executed := make(map[string]bool)
    maxIterations := len(config.Nodes) + 5

    for iteration := 0; iteration < maxIterations; iteration++ {
        allDone := true
        
        for _, node := range config.Nodes {
            if executed[node.ID] {
                continue
            }
            
            // 检查前置依赖是否完成
            canExecute := true
            for _, edge := range config.Edges {
                if edge.To == node.ID {
                    if !executed[edge.From] {
                        canExecute = false
                        break
                    }
                }
            }
            
            if !canExecute {
                allDone = false
                continue
            }
            
            // 执行节点
            allDone = false
            success := h.executeNode(h.DB, pipeline, run, &node)
            executed[node.ID] = success
        }
    }
}

#### 1.2.3 任务类型分类

**本版本只实现以下 4 种任务类型**：

| 任务类型 | 执行方式 | 说明 |
|---------|---------|------|
| **shell** | Agent | 执行自定义 Shell 脚本命令 |
| **docker** | Agent | Docker 镜像构建操作 |
| **git_clone** | Agent | Git 代码仓库检出 |
| **email** | Server | 邮件通知发送 |

**执行方式说明**：
- **Agent 执行**: 需要执行命令的任务（shell/docker/git_clone）
- **Server 执行**: 只需要调用 API 的任务（email）

**为什么代码仓库信息需要绑定到任务级别？**

因为一个流水线可能包含来自**不同代码仓库**的任务：

```

│                                                              │

**典型场景**：
| 场景 | 前端仓库 | 后端仓库 | 配置仓库 |
|------|---------|---------|---------|

| 多服务部署 | service-a | service-b | helm-charts |

**设计优势**：
1. 每个任务可独立指定仓库和分支
2. 支持不同仓库使用不同凭据
3. 任务执行时可追溯到具体代码版本
4. 支持同一个流水线触发多个仓库的 webhook

##### 1.2.4.6 变量实现机制

**变量解析时机**：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    变量生命周期与解析时机                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Phase 1: 流水线启动时                                                   │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │  • 解析 PipelineConfig 中的 ${inputs.*} 引用                       │   │
│  │  • 从用户输入或默认配置中获取 inputs 值                              │   │
│  │  • 注入到 PipelineRun 上下文                                       │   │
│  └──────────────────────────────────────────────────────────────────┘   │
│                                │                                         │
│                                ▼                                         │
│  Phase 2: 任务调度前（每个任务执行前）                                    │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │  • 解析 ${outputs.<node_id>.*} 引用                                │   │
│  │  • 从已完成的上游任务获取 outputs                                   │   │
│  │  • 解析 ${env.*} 引用，生成环境变量                                 │   │

#### 2.2.1 DAG 执行引擎 (新增文件)

**文件**: `easydo-server/internal/executor/dag_engine.go`

**伪代码风格**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DAGEngine:
  nodes: Map<NodeID, PipelineNode>       // 节点映射

  levels: Map<NodeID, Integer>           // 执行层级
  runID: Integer                         // PipelineRun ID

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FUNCTION NewDAGEngine(config: PipelineConfig, runID: Integer, db: Database) → DAGEngine:
  engine ← NEW DAGEngine
  engine.nodes ← NEW Map
  engine.edges ← NEW Map
  engine.inDegree ← NEW Map
  engine.levels ← NEW Map
  engine.runID ← runID
  engine.db ← db
  
  // 初始化节点
  FOR node IN config.nodes:
    engine.nodes[node.id] ← node
    engine.inDegree[node.id] ← 0
  
  // 构建邻接表和入度
  FOR edge IN config.edges:
    engine.edges[edge.from].APPEND(edge.to)
    engine.inDegree[edge.to] ← engine.inDegree[edge.to] + 1
  
  RETURN engine

────────────────────────────────────────────────────────────────

FUNCTION ValidateDependency() → ExecutionPlanData:
  // 1. 检测循环依赖
  IF engine.detectCycle() HAS error:
    RETURN error
  
  // 2. 计算执行层级（BFS）
  engine.calculateLevels()
  
  // 3. 构建执行计划
  RETURN engine.buildExecutionPlan()

────────────────────────────────────────────────────────────────

FUNCTION detectCycle() → error:
  visited ← NEW Map<NodeID, Integer>  // 0:unvisited, 1:visiting, 2:done
  recursionStack ← NEW Set<NodeID>
  
  FUNCTION dfs(nodeID):
    IF nodeID IN recursionStack:
      RETURN error("cycle detected: " + nodeID)
    
    IF visited[nodeID] = 2:
      RETURN null
    
    visited[nodeID] ← 1
    recursionStack.ADD(nodeID)
    
    FOR dependent IN engine.edges[nodeID]:
      IF dfs(dependent) HAS error:
        RETURN error
    
    visited[nodeID] ← 2
    recursionStack.DELETE(nodeID)
    RETURN null
  
  FOR nodeID IN engine.nodes:
    IF visited[nodeID] = 0:
      IF dfs(nodeID) HAS error:
        RETURN error
  
  RETURN null

────────────────────────────────────────────────────────────────

FUNCTION calculateLevels():
  // Kahn's Algorithm: BFS 计算层级
  queue ← NEW Queue<NodeID>
  
  FOR nodeID, degree IN engine.inDegree:
    IF degree = 0:
      engine.levels[nodeID] ← 0
      queue.ENQUEUE(nodeID)
  
  WHILE queue NOT EMPTY:
    current ← queue.DEQUEUE()
    
    FOR dependent IN engine.edges[current]:
      engine.inDegree[dependent] ← engine.inDegree[dependent] - 1
      
      IF engine.inDegree[dependent] = 0:
        engine.levels[dependent] ← engine.levels[current] + 1
        queue.ENQUEUE(dependent)

────────────────────────────────────────────────────────────────

FUNCTION buildExecutionPlan() → ExecutionPlanData:
  plan ← NEW ExecutionPlanData
  plan.totalNodes ← engine.nodes.LENGTH
  plan.maxLevel ← 0
  plan.levels ← NEW List
  plan.tasks ← NEW List
  
  // 按层级组织节点
  levelMap ← NEW Map<Integer, List<NodeID>>
  
  FOR nodeID, level IN engine.levels:
    levelMap[level].APPEND(nodeID)
    plan.maxLevel ← MAX(plan.maxLevel, level)
  
  // 构建层级信息
  FOR level FROM 0 TO plan.maxLevel:
    nodeIDs ← levelMap[level]  (SORTED)
    
    levelInfo ← NEW ExecutionLevel
    levelInfo.level ← level
    levelInfo.nodeIDs ← nodeIDs
    levelInfo.canRun ← true
    plan.levels.APPEND(levelInfo)
    
    // 构建任务信息
    FOR nodeID IN nodeIDs:
      node ← engine.nodes[nodeID]
      
      task ← NEW ExecutionTask
      task.nodeID ← nodeID
      task.name ← node.name
      task.level ← level
      task.dependencies ← engine.edges[nodeID]
      task.status ← "pending"
      plan.tasks.APPEND(task)
  
  RETURN plan
```

**实现说明**：

| 算法 | 时间复杂度 | 说明 |
|------|-----------|------|
| detectCycle | O(V+E) | DFS 检测循环依赖 |
| calculateLevels | O(V+E) | Kahn's Algorithm BFS |
| buildExecutionPlan | O(V+E) | 构建执行计划 |

// CanExecute 检查节点是否可以执行
func (e *DAGEngine) CanExecute(nodeID string, completedTasks map[string]bool) bool {
    // 检查所有前置依赖是否完成
    for _, dependent := range e.edges[nodeID] {
        if !completedTasks[dependent] {
            return false
        }
    }
    return true
}

// GetExecutableNodes 获取可执行节点列表
func (e *DAGEngine) GetExecutableNodes(completedTasks map[string]bool) []string {
    executable := make([]string, 0)
    
    for nodeID := range e.nodes {
        if completedTasks[nodeID] {
            continue
        }
        
        if e.CanExecute(nodeID, completedTasks) {
            executable = append(executable, nodeID)
        }
    }
    
    return executable
}

FUNCTION Evaluate(condition: String) → Boolean:
  // 求值条件表达式主入口
  
  // Step 1: 空条件或 "true" 直接返回成功
  IF condition IS_EMPTY OR condition == "true":
    RETURN true
  
  // Step 2: 解析变量引用
  // ${outputs.node1.field} → 实际值
  condition ← resolveVariables(condition)
  
  // Step 3: 处理操作符
  // "==" → "=", "!=" → " != "
  // "&&" → " && ", "||" → " || "
  condition ← processOperators(condition)
  
  // Step 4: 安全求值（避免代码注入）
  RETURN safeEval(condition)

FUNCTION resolveVariables(condition: String) → String:
  // 解析 ${...} 格式的变量引用
  
  // 使用正则匹配 ${...} 模式
  matches ← FIND_ALL_REGEX(condition, r'\$\{([^}]+)\}')
  
  FOR match IN matches:
    // 提取变量路径，如 "outputs.node1.status"
    paramPath ← EXTRACT_GROUP(match, 1)
    
    // 获取变量实际值
    value ← getValue(paramPath)
    
    IF value IS error:
      // 变量不存在时返回 false，使条件不满足
      replacement ← "false"
    ELSE:
      // 转换为字符串常量
      replacement ← STRINGIFY(value)
    
    // 替换原变量引用
    condition ← REPLACE(condition, match, replacement)
  
  RETURN condition

FUNCTION getValue(path: String) → Any:
  // 从上下文中获取变量值
  
  // 按 "." 分割路径
  parts ← SPLIT(path, ".")
  
  IF parts.LENGTH < 2:
    RETURN error("invalid path: " + path)
  
  // 根据前缀选择获取方式
  prefix ← parts[0]
  
  SWITCH prefix:
    CASE "outputs":
      RETURN getOutputValue(parts[1:])
    CASE "inputs":
      RETURN getInputValue(parts[1:])
    CASE "env":
      RETURN getEnvValue(parts[1:])
    DEFAULT:
      RETURN error("unknown prefix: " + prefix)

FUNCTION processOperators(condition: String) → String:
  // 标准化操作符格式
  
  // 比较操作符
  condition ← REPLACE_ALL(condition, "==", "=")
  condition ← REPLACE_ALL(condition, "!=", " != ")
  
  // 逻辑操作符（添加空格便于解析）
  condition ← REPLACE_ALL(condition, "&&", " && ")
  condition ← REPLACE_ALL(condition, "||", " || ")
  
  RETURN condition

FUNCTION safeEval(condition: String) → Boolean:
  // 安全求值表达式（递归下降解析）
  
  condition ← TRIM(condition)
  
  // Step 1: 处理括号（从内向外递归）
  WHILE condition CONTAINS "(":
    // 找到最内层括号
    openIndex ← LAST_INDEX_OF(condition, "(")
    closeIndex ← INDEX_OF(condition, ")", openIndex)
    
    IF closeIndex == -1:
      RETURN error("mismatched parentheses")
    
    // 提取括号内表达式
    innerExpr ← SUBSTRING(condition, openIndex + 1, closeIndex - 1)
    
    // 递归求值
    result ← safeEval(innerExpr)
    
    // 替换为常量
    IF result == true:
      condition ← REPLACE_RANGE(condition, openIndex, closeIndex + 1, "true")
    ELSE:
      condition ← REPLACE_RANGE(condition, openIndex, closeIndex + 1, "false")
  
  // Step 2: 处理 AND (&&)
  IF condition CONTAINS "&&":
    parts ← SPLIT(condition, "&&")
    
    FOR part IN parts:
      part ← TRIM(part)
      result ← evalPrimitive(part)
      
      IF result IS false:
        RETURN false  // AND 中一个 false 结果为 false
    
    RETURN true
  
  // Step 3: 处理 OR (||)
  IF condition CONTAINS "||":
    parts ← SPLIT(condition, "||")
    
    FOR part IN parts:
      part ← TRIM(part)
      result ← evalPrimitive(part)
      
      IF result IS true:
        RETURN true  // OR 中一个 true 结果为 true
    
    RETURN false
  
  // Step 4: 处理基本表达式
  RETURN evalPrimitive(condition)

────────────────────────────────────────────────────────────────

FUNCTION evalPrimitive(condition: String) → Boolean:
  // 求值基本比较表达式
  // 格式: <value1> <operator> <value2>
  
  condition ← TRIM(condition)
  
  // 处理字面量
  IF condition == "true":
    RETURN true
  IF condition == "false":
    RETURN false
  
  // 解析比较表达式
  tokens ← SPLIT_BY_SPACE(condition)
  
  IF tokens.LENGTH != 3:
    RETURN error("invalid expression: " + condition)
  
  left ← UNQUOTE(tokens[0])
  op ← tokens[1]
  right ← UNQUOTE(tokens[2])
  
  SWITCH op:
    CASE "=":
      RETURN left == right
    CASE "!=":
      RETURN left != right
    CASE ">":
      RETURN compareNumbers(left, right, (a, b) → a > b)
    CASE ">=":
      RETURN compareNumbers(left, right, (a, b) → a >= b)
    CASE "<":
      RETURN compareNumbers(left, right, (a, b) → a < b)
    CASE "<=":
      RETURN compareNumbers(left, right, (a, b) → a <= b)
    DEFAULT:
      RETURN error("unknown operator: " + op)

────────────────────────────────────────────────────────────────

FUNCTION compareNumbers(left: String, right: String, 
                       op: Function<Number, Number, Boolean>) → Boolean:
  // 数字比较（支持字符串和数字）
  
  numLeft ← PARSE_NUMBER(left)
  numRight ← PARSE_NUMBER(right)
  
  IF numLeft IS success AND numRight IS success:
    // 两者都是数字，进行数字比较
    RETURN op(numLeft, numRight)
  ELSE:
    // 否则进行字符串比较
    RETURN op(LENGTH(left), LENGTH(right))
```

                return false, nil
            }
        }

    if strings.Contains(condition, "||") {

            }

// evalPrimitive 求值原始表达式
func (ce *ConditionEvaluator) evalPrimitive(condition string) (bool, error) {
    condition = strings.TrimSpace(condition)
    
    if condition == "true" {
        return true, nil
    }
    if condition == "false" {
        return false, nil
    }
    
    // 解析比较表达式
    parts := strings.Fields(condition)
    if len(parts) != 3 {
        return false, fmt.Errorf("invalid expression: %s", condition)
    }
    
    left, op, right := parts[0], parts[1], parts[2]
    
    // 去除字符串引号
    left = strings.Trim(left, "\"")
    right = strings.Trim(right, "\"")
    
    switch op {
    case "=":
        return left == right, nil
    case "!=":
        return left != right, nil
    case ">":
        return ce.compareNumbers(left, right, func(a, b float64) bool { return a > b })
    case ">=":
        return ce.compareNumbers(left, right, func(a, b float64) bool { return a >= b })
    case "<":
        return ce.compareNumbers(left, right, func(a, b float64) bool { return a < b })
    case "<=":
        return ce.compareNumbers(left, right, func(a, b float64) bool { return a <= b })
    default:
        return false, fmt.Errorf("unknown operator: %s", op)
    }
}

// compareNumbers 比较数字
func (ce *ConditionEvaluator) compareNumbers(left, right string, op func(a, b float64) bool) (bool, error) {
    a, err1 := strconv.ParseFloat(left, 64)
    b, err2 := strconv.ParseFloat(right, 64)
    
    if err1 == nil && err2 == nil {
        return op(a, b), nil
    }
    
    // 字符串比较
    return op(float64(len(left)), float64(len(right))), nil
}

#### 2.2.2 单 Agent 绑定设计（关键架构决策）

**设计原则**：每次 PipelineRun 执行时，**只选择一个 Agent**，将所有任务分配给该 Agent。

FUNCTION selectAgentForPipelineRun(config: PipelineConfig) → Agent:
  // 为流水线执行选择最优 Agent
  
  // Step 1: 收集流水线所需的标签需求
  requiredLabels ← collectRequiredLabels(config)
  
  // Step 2: 查询候选 Agent
  candidates ← NEW List<Agent>
  
  // 数据库查询条件:
  // - status IN (online, busy)
  // - registration_status = approved
  // - labels 包含所有 requiredLabels
  FOR label IN requiredLabels:
    query ← query.AND("labels LIKE %" + label + "%")
  
  candidates ← DB.Find(Agent, query)
  
  IF candidates.IS_EMPTY:
    RETURN error("no available agent matching labels: " + JOIN(requiredLabels, ","))
  
  // Step 3: 负载均衡选择（评分制）
  bestAgent ← candidates[0]
  bestScore ← calculateAgentScore(bestAgent)
  
  FOR i FROM 1 TO candidates.LENGTH - 1:
    score ← calculateAgentScore(candidates[i])
    
    IF score > bestScore:
      bestScore ← score
      bestAgent ← candidates[i]
  
  RETURN bestAgent

────────────────────────────────────────────────────────────────

FUNCTION calculateAgentScore(agent: Agent) → Number:
  // 计算 Agent 评分（分数越高越优先）
  
  // 初始分数
  score ← 100.0
  
  // 负载因素：正在运行的任务越少，分数越高
  // 每个运行中任务扣 15 分
  score ← score - (agent.tasksRunning * 15.0)
  
  // 资源因素：剩余内存越多分数越高
  // 内存使用率越高，扣分越多（最高扣 30 分）
  IF agent.memoryTotal > 0 AND agent.memoryFree > 0:
    usedPercent ← (agent.memoryTotal - agent.memoryFree) / agent.memoryTotal
    score ← score - (usedPercent * 30.0)
  
  // 健康因素：连续成功次数越多分数越高
  // 每次成功加 5 分
  score ← score + (agent.consecutiveSuccess * 5.0)
  
  RETURN score

────────────────────────────────────────────────────────────────

FUNCTION collectRequiredLabels(config: PipelineConfig) → List<String>:
  // 从流水线配置中收集所有标签需求
  
  labelSet ← NEW Set<String>
  
  FOR node IN config.nodes:
    IF node.labels IS NOT null:
      FOR label IN node.labels:
        labelSet.ADD(label)
  
  RETURN labelSet.TO_LIST()
```

     = 86

Agent B: 运行中任务=0, 内存使用率=80%, 连续成功=2

// executePipelineTasksEnhanced 增强版任务执行（支持DAG依赖 + 配置快照）
func (h *PipelineHandler) executePipelineTasksEnhanced(pipeline models.Pipeline, run *models.PipelineRun) {
    // 1. 解析流水线配置（从 Pipeline 表获取最新配置）
    var config PipelineConfig
    if err := json.Unmarshal([]byte(pipeline.Config), &config); err != nil {
        h.updateRunStatus(run.ID, "failed", "解析流水线配置失败: "+err.Error())
        return
    }

    // 2. 关键步骤：保存配置快照到 PipelineRun
    // 因为流水线可能被编辑，必须保存执行时的配置副本
    run.Config = pipeline.Config
    h.DB.Model(run).Update("config", run.Config)

    // 3. 选择执行 Agent（一次 PipelineRun 只使用一个 Agent）
    selectedAgent, err := h.selectAgentForPipelineRun(&config)
    if err != nil {
        h.updateRunStatus(run.ID, "failed", "选择执行器失败: "+err.Error())
        return
    }
    run.AgentID = selectedAgent.ID
    h.DB.Model(run).Update("agent_id", run.AgentID)

    // 4. 创建DAG引擎
    engine := executor.NewDAGEngine(&config, run.ID, h.DB)

    // 5. 验证依赖并生成执行计划
    plan, err := engine.ValidateDependency()
    if err != nil {
        h.updateRunStatus(run.ID, "failed", "依赖验证失败: "+err.Error())
        return
    }

    // 6. 保存执行计划
    planData, _ := json.Marshal(plan)
    executionPlan := &models.PipelineExecutionPlan{
        PipelineRunID: run.ID,
        PlanData:      string(planData),
        Status:        "building",
        StartedAt:     time.Now().Unix(),
    }
    h.DB.Create(executionPlan)

    // 7. 更新运行状态
    h.updateRunStatus(run.ID, "running", "")

    // 8. 按层级执行任务
    completedTasks := make(map[string]bool)
    taskResults := make(map[string]map[string]interface{})  // nodeID -> outputs

    for level := 0; level <= plan.MaxLevel; level++ {
        // 获取当前层级的所有可执行任务
        levelInfo := plan.Levels[level]
        
        // 并行执行当前层级的任务
        var wg sync.WaitGroup
        taskChan := make(chan string, len(levelInfo.NodeIDs))
        
        for _, nodeID := range levelInfo.NodeIDs {
            wg.Add(1)
            taskChan <- nodeID
        }
        
        close(taskChan)
        
        // 并发执行
        for nodeID := range taskChan {
            go func(nodeID string) {
                defer wg.Done()
                
                node := config.Nodes[0]
                for _, n := range config.Nodes {
                    if n.ID == nodeID {
                        node = n
                        break
                    }
                }

                // 检查条件
                if node.When != "" {
                    evaluator := executor.NewConditionEvaluator()
                    evaluator.SetContext(map[string]interface{}{
                        "outputs": taskResults,
                        "inputs":  map[string]interface{}{},  // 流水线输入
                        "env":     map[string]string{},
                    })
                    
                    canRun, err := evaluator.Evaluate(node.When)
                    if err != nil || !canRun {
                        // 条件不满足，跳过任务
                        completedTasks[nodeID] = true
                        taskResults[nodeID] = map[string]interface{}{
                            "status": "skipped",
                            "reason": "condition_not_met",
                        }
                        return
                    }
                }

                // 执行任务
                success := h.executeNodeWithDependency(h.DB, pipeline, run, &node, completedTasks)
                completedTasks[nodeID] = success
                
                // 收集任务输出
                if success {
                    taskResults[nodeID] = map[string]interface{}{
                        "status":   "success",
                        "exitcode": 0,
                        "outputs":  h.getNodeOutputs(run.ID, nodeID),
                    }
                } else {
                    taskResults[nodeID] = map[string]interface{}{
                        "status": "failed",
                        "reason": "execution_failed",
                    }
                }
            }(nodeID)
        }
        
        wg.Wait()
        
        // 检查是否有失败且不允许继续的任务
        allSuccess := true
        for _, nodeID := range levelInfo.NodeIDs {
            if !completedTasks[nodeID] {
                allSuccess = false
                break
            }
        }
        
        if !allSuccess {
            h.updateRunStatus(run.ID, "failed", "层级"+strconv.Itoa(level)+"执行失败")
            executionPlan.Status = "failed"
            executionPlan.FinishedAt = time.Now().Unix()
            h.DB.Save(executionPlan)
            return
        }
    }

    // 7. 所有任务完成
    h.updateRunStatus(run.ID, "success", "")
    executionPlan.Status = "success"
    executionPlan.FinishedAt = time.Now().Unix()
    h.DB.Save(executionPlan)
}

// executeNodeWithDependency 执行单个节点（修正版）
// 核心原则：
// 1. 执行类任务（shell/docker/script等）→ 发送给 Agent 执行
// 2. 通知类任务（webhook/email）→ Server 直接执行
// 3. 控制类任务（condition/loop）→ Server 处理逻辑
func (h *PipelineHandler) executeNodeWithDependency(db *gorm.DB, pipeline models.Pipeline, run *models.PipelineRun, node *models.PipelineNode, completedTasks map[string]bool) bool {
    
    // 1. 检查是否为控制类任务（不需要实际执行）
    if node.IsServerTask() && !node.IsAgentTask() {
        return h.executeServerTask(node, completedTasks)
    }
    
    // 2. 执行类任务 → 发送给 Agent
    if node.IsAgentTask() {
        return h.executeAgentTask(db, pipeline, run, node)
    }
    
    // 未知类型，默认交给 Agent 执行
    return h.executeAgentTask(db, pipeline, run, node)
}

// executeAgentTask 执行 Agent 任务（修正版）
func (h *PipelineHandler) executeAgentTask(db *gorm.DB, pipeline models.Pipeline, run *models.PipelineRun, node *models.PipelineNode) bool {
    // 1. 选择可用的 Agent
    var agents []models.Agent
    db.Where("status = ? AND registration_status = ?",
        models.AgentStatusOnline, models.AgentRegistrationStatusApproved).Find(&agents)
    
    if len(agents) == 0 {
        h.logError(run.ID, "no available agents")
        return false
    }
    
    // 选择负载最低的 Agent
    agentID := agents[0].ID
    minTasks := int64(999999)
    
    for _, agent := range agents {
        var runningTasks int64
        db.Model(&models.AgentTask{}).Where("agent_id = ? AND status = ?", agent.ID, models.TaskStatusRunning).Count(&runningTasks)
        if runningTasks < minTasks {
            minTasks = runningTasks
            agentID = agent.ID
        }
    }
    
    // 2. 提取任务参数
    script := ""
    timeout := 3600
    
    // 从 params 中提取命令/脚本
    if cmd, ok := node.Params["command"].(string); ok {
        script = cmd
    } else if cmd, ok := node.Params["script"].(string); ok {
        script = cmd
    }
    
    if t, ok := node.Params["timeout"].(float64); ok {
        timeout = int(t)
    }
    
    // 确定 TaskType（映射到 Agent 认识的任务类型）
    taskType := mapNodeTypeToTaskType(node.Type)
    
    // 3. 提取仓库信息
    repoURL, repoBranch, repoCommit, repoPath := extractRepositoryInfo(node)
    
    // 4. 创建 AgentTask
    task := &models.AgentTask{
        AgentID:       agentID,
        PipelineRunID: run.ID,
        NodeID:        node.ID,
        TaskType:      taskType,
        Name:          node.Name,
        Script:        script,
        Status:        models.TaskStatusPending,
        Timeout:       timeout,
        Params:        encodeParams(node.Params),
        // 仓库信息
        RepoURL:       repoURL,
        RepoBranch:    repoBranch,
        RepoCommit:    repoCommit,
        RepoPath:      repoPath,
    }
    
    if err := db.Create(task).Error; err != nil {
        h.logError(run.ID, "failed to create task: "+err.Error())
        return false
    }
    
    // 5. 等待任务完成
    return h.waitForTaskCompletion(db, run.ID, task.ID, timeout)
}

// waitForTaskCompletion 等待任务完成
func (h *PipelineHandler) waitForTaskCompletion(db *gorm.DB, runID, taskID uint64, timeout int) bool {
    timeoutChan := time.After(time.Duration(timeout+30) * time.Second)
    tickChan := time.Tick(5 * time.Second)
    
    for {
        select {
        case <-timeoutChan:
            db.Model(&models.AgentTask{}).Where("id = ?", taskID).Updates(map[string]interface{}{
                "status":    models.TaskStatusFailed,
                "error_msg": "任务执行超时",
                "end_time":  time.Now().Unix(),
            })
            return false
            
        case <-tickChan:
            var task models.AgentTask
            db.First(&task, taskID)
            
            switch task.Status {
            case models.TaskStatusSuccess:
                return true
            case models.TaskStatusFailed:
                return false
            case models.TaskStatusCancelled:
                return false
            }
        }
    }
}

// executeWebhook 执行 Webhook 任务
func (h *PipelineHandler) executeWebhook(node *models.PipelineNode) bool {

    url, _ := node.Params["url"].(string)

    if method == "" {
        method = "POST"
    }
    headers := extractHeaders(node.Params["headers"])
    body := node.Params["body"]
    
    // 执行 HTTP 请求
    client := &http.Client{Timeout: 30 * time.Second}
    req, err := http.NewRequest(method, url, nil)
    if err != nil {
        return false
    }
    
    for k, v := range headers {
        req.Header.Set(k, v)
    }
    
    if body != nil {
        jsonBody, _ := json.Marshal(body)
        req.Body = io.NopCloser(bytes.NewReader(jsonBody))
        req.Header.Set("Content-Type", "application/json")
    }
    
    resp, err := client.Do(req)
    if err != nil {
        return false
    }
    defer resp.Body.Close()
    
    return resp.StatusCode >= 200 && resp.StatusCode < 400
}

    id := c.Param("id")
    
    var pipeline models.Pipeline

        c.JSON(http.StatusNotFound, gin.H{
            "code":    404,
            "message": "流水线不存在",
        })
        return
    }
    
    // 解析配置
    var config PipelineConfig
    if err := json.Unmarshal([]byte(pipeline.Config), &config); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,
            "message": "流水线配置解析失败",
        })
        return
    }
    
    // 创建DAG引擎验证
    engine := executor.NewDAGEngine(&config, 0, nil)
    plan, err := engine.ValidateDependency()
    
    if err != nil {
        c.JSON(http.StatusOK, gin.H{
            "code": 200,
            "data": gin.H{
                "valid":          false,
                "error":          err.Error(),
                "error_type":     "cycle_detected",
            },
        })
        return
    }
    
    c.JSON(http.StatusOK, gin.H{
        "code": 200,
        "data": gin.H{
            "valid":       true,
            "total_nodes": plan.TotalNodes,
            "max_level":   plan.MaxLevel,
            "levels":      plan.Levels,
        },
    })
}

## 七、WebSocket 通信架构

> **注意**: WebSocket 通信架构的详细设计已移至独立文档 [doc/f01_websocket.md](f01_websocket.md)

### 7.1 架构概述

WebSocket 通信用于实现以下场景的实时交互：

| 通信方向 | 用途 | 消息类型 |
|----------|------|----------|
| Server → Agent | 任务下发 | task_assign, task_cancel, agent_config |
| Agent → Server | 日志上报 | task_log, task_status, heartbeat |
| Server → Frontend | 实时状态 | task_status, task_log, run_progress, agent_status |
| Frontend → Server | 订阅管理 | subscribe, unsubscribe |

### 7.2 通信协议概览

#### 7.2.1 Server ↔ Agent 消息

| 方向 | 消息类型 | 描述 |
|------|----------|------|
| Server → Agent | task_assign | 下发任务 |
| Server → Agent | task_cancel | 取消任务 |
| Agent → Server | task_log | 上报日志 |
| Agent → Server | task_status | 上报状态 |
| 双向 | heartbeat | 心跳保活 |

### 8.5 设计简化记录

| 原始设计 | 简化后 | 状态 | 说明 |
|----------|--------|------|------|
| Connections | Edges | ✅ 已完成 | 命名更简洁 |
| PipelineNodeGroup | 移除 | ✅ 已完成 | DAG层级自动确定并行度 |
| PipelineInput | 合并到 Settings | ⏳ 待定 | 可选配置 |
| PipelineOutput | 合并到 Settings | ⏳ 待定 | 可选配置 |
| PipelineExecutionPlan | 动态计算 | ⏳ 待定 | 无需持久化 |
| TaskDependency | 从 Edges 推导 | ⏳ 待定 | 冗余存储 |

### 8.6 核心数据模型关系图

```
Pipeline (模板)
